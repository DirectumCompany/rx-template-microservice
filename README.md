# Микросервис с API интерфейсом
Шаблон заказного микросервиса с API интерфейсом

Инструмент закрывает следующие требования:
* вынесение на отдельный сервис заведомо опасной отделяемой функциональности, чтобы при её сбое не был затронут основной сервис (Directum RX);
* реализация мини-фичи в виде отдельного сервиса, когда она отсутствует в платформе или её реализация средствами платформы обходится слишком дорого.

В решении 2 проекта:
* CustomWebAPI - ASP .NET Core веб-приложение на Kestrel с REST API (пример CRUD-контроллера), порт задаётся настройкой PORT; можно использовать шаблон для реализации сайтов и веб-сервисов
* CustomJobHandler - консольное Worker Service-приложение, выполняющее фоновую задачу с периодичностью, задаваемой настройкой INTERVAL; можно использовать для реализации легковесных гибко настраиваемых агентов, не требующих обработки запросов

> [!WARNING]
> Актуально только для Directum RX версий до 4.6 включительно. Начиная с Directum RX 4.6 задачи, которые решал данный шаблон, закрывает штатный «Сервис изолированного кода» (IsolatedArea) — он отвечает за выполнение стороннего/прикладного кода в отдельном изолированном процессе, чтобы ошибки в нём не влияли на остальные сервисы (веб-сервер, сервис асинхронных событий, сервис отчётов, сервис интеграции и др.). Для RX 4.6 и новее используйте штатный механизм вместо данного шаблона.

> [!NOTE]
> Решение предоставляется "как есть" (as is), без каких-либо гарантий и без технической поддержки.

## Порядок запуска микросервиса средствами ServiceRunner (только для Windows):
1. Выполнить публикацию проекта Self-Contained.
2. Содержимое заархивировать в zip, назвать <sevice_name>.zip, рекомендуется назвать аналогично проекту, иначе предварительно поправить имя в _PackageInfo.json.
3. Скопировать архив в \etc\_builds_package
4. На основе _ConfigSettings.xml.example создать _ConfigSettings.xml в папке \etc\_services_config\<sevice_name>
5. Добавить в конфиге ServiceRunner'а (\etc\_services_config\ServiceRunner) строку:
<ServiceSetting Name="<sevice_name>" Config="<sevice_name>\_ConfigSettings.xml" Package="<sevice_name>.zip" ConfigWatcherEnabled="false" Disabled="false" />
